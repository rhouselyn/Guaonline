技术核心 · TOOL CALLING

用 LLM 自带的工具调用，
把非结构化文本变成结构化数据。

⓵

工具调用 = 结构化输出

不靠正则解析 LLM 文本，而是用模型原生 tool_calls 能力，让 LLM 直接返回 JSON 结构——词、释义、音标、例句、题干、选项全部字段化，零解析失败。

⓶

程序只做编排存储

拿到结构化结果后，程序负责存储、跳转、收藏、组卷。单词点击 0 等待跳转，信息密度高度集中。

⓷

Token 利用率极致

工具调用 + 紧凑字段命名 + 自动分句切分，token 消耗降至最低。这也是能把单价压到 $4.8/1M 的关键。

// LLM TOOL_CALL OUTPUT
// 用户输入
"He abandoned the plan."
// LLM 工具调用返回
tool_call: parse_sentence
args: {
  sentence: "He abandoned the plan.",
  translation: "他放弃了这个计划。",
  words: [
    { w: "abandon",
      pos: "v.",
      phon: "/əˈbændən/",
      meaning: "放弃；废弃",
      example: "She abandoned..." },
    { w: "plan", ... }
  ],
  grammar: "主谓宾结构..."
}
// 程序：存储 + 渲染 + 组卷
store(result) → render() → generate_exercises()
