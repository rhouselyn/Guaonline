## 1. 执行摘要

**呱邻国(Gualingo)** 是一个 AI 原生的沉浸式外语学习平台。用户投入任意一段文本(新闻、歌词、合同、论文,或一句"我想学攀岩相关法语"),LLM 通过原生工具调用(Tool Calling)现场完成分句、翻译、拆词、词性、音标、语法解释、例句生成与五种题型派生,把"任何一段文字"变成一套可学习、可复习、可测验的专属课程。

**核心差异化:**
- **任意文本** —— 不靠预制题库,内容来自用户自带素材或 AI 生成,覆盖长尾兴趣/专业场景(外文歌、攀岩术语、外文合同)。
- **200+ 语言** —— 任意语种学任意语种,界面也可任意切换;**新增一门语言 = 0 行代码**,只要模型会,用户就能学。
- **10 倍成本优势** —— 同等 LLM 能力,通用翻译 API 收 $48/1M tokens,符合条件的能力仅 $4.8/1M;通过工具调用 + 紧凑字段 + 自动分句,把 token 消耗压到最低。
- **结构化输出零解析失败** —— 不靠正则解析 LLM 文本,直接用模型原生 tool_calls 返回 JSON,程序只做编排存储,单词点击 0 等待跳转。

**商业模式:Freemium SaaS。** 社区版(开源 AGPL v3,BYOK 自助)做口碑与冷启动;商业版(闭源 SaaS)卖"便利 + 效果"—— 平台 API 代充、云同步、SRS 间隔复习、AI 口语、学习分析。订阅三档:免费 / 基础(¥19-29/月)/ 专业(¥49-69/月)。留存机制:每日打卡 → 系统闲时预生成明日内容 → 打开即学(0 等待)→ 形成依赖,LLM 成本摊薄到非高峰时段,毛利可控。

**当前阶段:** 核心学习引擎、JWT 认证、管理后台、LLM 网关多 Key 轮询、配额系统、多语言 UI、5 种题型、TTS 语音均已落地并有测试覆盖;处于内测,尚未正式发布。

**融资计划 `[待校准]`:** 拟寻求种子轮 **¥[80万-150万]**,出让 **[10%-15%]** 股权,资金用于 12 个月内完成 SaaS 上线、首批 1 万月活与付费转化验证,达到可启动下一轮的 PMF 指标。

## 3. 问题与市场

### 3.1 用户痛点

**大众语种(如日语)有专门 App,小语种几乎无人覆盖 —— 市场缺口巨大,且注定要靠 LLM 灵活性而非预制数据库来填。**

| 场景 | 痛点 | 现有方案的失败 |
|------|------|----------------|
| 兴趣驱动 | 想学会一首外文歌每一句歌词、每个生词的语境含义 | 多邻国没有这条路径 |
| 小众场景 | 攀岩、潜水、滑雪等专业术语 | 数据库预制永远追不上长尾 |
| 应试/实用 | 中学生完全学会一篇阅读理解;职场人吃透一份外文合同 | 这才是"针对性学习" |
| 进阶乏力 | 学到 B1 后词汇重复高 | 预制内容迭代慢 |
| 翻译≠学习 | 复制→翻译→离开,不产生学习行为 | 翻译软件是工具,不是学习系统 |

### 3.2 市场规模 `[待校准]`

- 全球语言学习市场:约 **¥[600-800 亿]** (2025),CAGR ~15-18%,AI 驱动产品增速显著高于整体。
- 多邻国 2024 年营收约 **$748M**,月活 5 亿+,验证了"游戏化 + 多语种"的天花板与付费意愿。
- 增量机会在:**小语种长尾 + 自带内容 + AI 生成** 三条主线,这是预制数据库模型无法覆盖的空白。

### 3.3 目标用户(按优先级)

1. **学生群体(中学/大学)** —— 阅读理解、考试备考、留学小语种。手机使用率高,付费意愿中等但基数大。**优先。**
2. **旅游人士** —— 实用场景对话学习,短周期、强即时需求。
3. **兴趣/专业学习者** —— 外文歌、影视、专业阅读,高黏性、高 LTV。
4. **教育机构 / B 端(远期)** —— 教师管理后台、班级作业、白标方案。

## 4. 产品与方案

### 4.1 核心理念

> **程序只提供骨架,内容全部交给 LLM。**

传统语言 App 靠预制内容库驱动,每加一门语言 = 多建一整套数据库,长尾需求永远追不上。呱邻国靠 **LLM 现场生成**驱动,每加一门语言 = **0 行代码**,只要模型会,用户就能学。

### 4.2 三种输入模式

1. **直接输入原文** —— 粘贴正在读的外文新闻、歌词、合同、论文,用户自带内容,AI 当场拆解。
2. **AI 翻译生成** —— 输入母语,LLM 翻成目标语言再拆解。没原文也能学,门槛降到零。
3. **AI 按主题生成** —— 说一句"我要学攀岩相关法语",LLM 直接产出对应文本,长尾场景也能覆盖。

### 4.3 处理流程(Pipeline)

```
接收文本(自动检测语种)
   ↓
LLM 工具调用:分句 / 翻译 / 拆词 / 词性 / 音标 / 语法
   ↓
结构化存储:JSON 字段化落库,单词可点击跳转
   ↓
编排渲染:卡片 / 释义 / 例句 / 收藏夹界面化
   ↓
自动出题:5 种题型从同一文本派生,闭环复习
```

### 4.4 技术核心:Tool Calling

不靠正则解析 LLM 文本,而是用**模型原生 tool_calls** 能力,让 LLM 直接返回 **JSON 结构** —— 词、释义、音标、例句、题干、选项全部字段化,**零解析失败**。拿到结构化结果后,程序负责存储、跳转、收藏、组卷。单词点击 0 等待跳转,信息密度高度集中。工具调用 + 紧凑字段命名 + 自动分句切分,把 token 消耗降至最低 —— 这是能把单价压到 $4.8/1M 的关键。

```jsonc
// LLM tool_call 输出示意
tool_call: parse_sentence
args: {
  sentence: "He abandoned the plan.",
  translation: "他放弃了这个计划。",
  words: [
    { w:"abandon", pos:"v.", phon:"/əˈbændən/",
      meaning:"放弃;废弃", example:"She abandoned..." },
    { w:"plan", ... }
  ],
  grammar: "主谓宾结构..."
}
// 程序:store(result) → render() → generate_exercises()
```

### 4.5 学习闭环

```
输入文本 → AI 分句翻译 → 生成词汇表
   → 阶段一:学单词(单词选择 / 句子翻译 / 听力理解)
   → 阶段二:练句子(遮蔽填空 / 翻译重组)
   → 错题回顾(自动收集,强化练习直到掌握)
```

五种题型从同一文本派生,形成闭环复习;收藏夹一键收薄弱点,循环复习。

## 5. 商业模式

### 5.1 双版本策略

```
┌─────────────────────────────────────────────┐
│       呱邻国 Community(开源 AGPL v3)         │
│  BYOK · 本地存储 · 多 Key 轮询 · 基础学习      │
│  Web 版 + 桌面端 · Docker 自部署              │
└──────────────────────┬──────────────────────┘
                       │ 共享核心代码
┌──────────────────────┴──────────────────────┐
│       呱邻国 Pro(闭源 SaaS)                  │
│  平台 API · 云同步 · SRS · 分析 · AI 口语     │
│  用户系统 · 支付 · 移动端 · 社交              │
└─────────────────────────────────────────────┘
```

- **社区版定位:** 不是"阉割版",而是"自助版"。功能完整,用户自管 API Key 和数据,是最好的口碑传播与社区贡献来源。AGPL v3 确保贡献回馈,防止闭源分支。
- **商业版价值:** 便利(云同步、平台 API、托管无需自部署)+ 效果(SRS、AI 口语、学习分析),不是功能封锁。

### 5.2 付费层级

| 层级 | 月费 | 包含 |
|------|------|------|
| **免费版** | ¥0 | BYOK,本地存储,基础学习功能(社区版) |
| **基础版** | ¥19-29 | 平台 API 额度(月 50 次文本处理)、云同步、SRS 间隔复习 |
| **专业版** | ¥49-69 | 无限 API 额度、AI 口语对话、学习分析、优先支持 |

### 5.3 三条变现逻辑

1. **API 代充(核心):** 批量采购 LLM API(SiliconFlow/DeepSeek 等),加价 30-50% 转售给用户。同等能力采购 $4.8/1M,通用翻译 API 售 $48/1M,中间有充足加价空间。
2. **云服务:** 云同步、跨设备、数据备份 —— 用户为便利付费。
3. **高级功能:** SRS、AI 口语、学习分析 —— 用户为效果付费。

### 5.4 留存与成本控制机制

> **每日打卡 → 系统闲时提前生成明日内容 → 用户打开即学(0 等待)→ 形成依赖 → 持续订阅。**

LLM 调用被摊薄到非高峰时段,既保证"打开即学"的体验,又把成本控制在闲时低负载,毛利可控。

## 6. 竞争分析

### 6.1 vs 多邻国(预制内容型)

| 维度 | 多邻国 | 呱邻国 |
|------|--------|--------|
| 内容来源 | 预制题库 | LLM 现场生成 / 用户自带 |
| 自定义文本 | ✗ | ✓ 任意文本 |
| 语种覆盖 | 主流 40+ | 200+,新增 0 行代码 |
| 针对性学习 | 弱,按固定轨迹 | 强,围绕用户内容 |
| 长尾/专业场景 | ✗ | ✓ |
| 游戏化留存 | 极强(连续打卡+排行榜) | 计划引入打卡+预生成 |
| 品牌/规模 | 5 亿+月活 | 0 → 待建 |

**多邻国的优点我们承认:** 游戏化机制留存极高、品牌碾压、题型打磨细致。**它的盲区是我们的机会:** 学不到想学的、小众语种不支持、无法处理兴趣/专业场景、进阶乏力。

### 6.2 vs 有道/通用翻译 API(翻译工具型)

| 维度 | 有道/通用翻译 API | 呱邻国 |
|------|-------------------|--------|
| Token 单价 | $48/1M | $4.8/1M(**10 倍价差**) |
| 支持语种 | 有限,主流 30-50 种 | 200+,任意互学 |
| 词汇拆解 | ✗ 只给整段翻译 | ✓ 按语种风格自动拆分 |
| 单词跳转 | ✗ 翻译完即结束 | ✓ 0 等待 |
| 例句/语法/题目生成 | ✗ 无 | ✓ 自动 |
| 行为属性 | 工具(用完即走) | 学习系统(产生学习行为) |

**结论:** 翻译软件是工具,呱邻国是学习系统。同样能力的 LLM,通用翻译 API 收 $48/1M,符合条件的能力只要 $4.8/1M —— 我们用更便宜的能力,做更深的拆解,且**只在我们这里产生学习行为**。

### 6.3 竞争壁垒

1. **架构壁垒:** "程序只做骨架,内容全部交给 LLM"的架构 + Tool Calling 结构化输出,是经过验证的工程范式,迁移成本高。
2. **成本壁垒:** Token 利用率极致优化带来的 10 倍成本优势,直接转化为定价空间与毛利。
3. **语种壁垒:** 200+ 语言任意互学、界面任意切换,同一套架构天生全球传播,预制数据库模型难以复制。
4. **开源壁垒:** AGPL v3 社区版做口碑与贡献来源,既获客又防止闭源分支分流。
5. **数据壁垒(远期):** 用户学习行为、错题、收藏、薄弱点数据积累,反哺 SRS 与内容生成质量。

## 11. 风险与应对

| 风险 | 等级 | 应对 |
|------|------|------|
| **API 成本不可控** | 高 | 每用户日/月上限 + 超限降级;批量采购压单价;闲时预生成;结果缓存复用 |
| **社区版分流付费用户** | 中 | 社区版定位"自助",商业版卖"便利+效果";AGPL 防闭源分流 |
| **多邻国等大平台竞争** | 高 | 差异化:任意文本、任意语言对、AI 生成内容;不正面打预制内容战 |
| **专业版"无限"亏损** | 中 | 软上限 + 降级提示;闲时预生成;高价值用户单独优化 |
| **LLM 质量不稳定** | 中 | 多模型切换、质量评估机制、缓存优质结果、回退策略 |
| **用户不愿付费** | 中 | 先验证 PMF;免费版足够好用以获客;付费版有明确升级动力 |
| **单人项目交付风险** | 中 | 融资后优先补全栈前端;关键模块已有测试覆盖 |
| **合规(数据/支付/出海)** | 中 | 用户数据隔离已实现;支付走合规通道;出海按 GDPR/区域合规 |
| **LLM 能力依赖第三方** | 中 | 网关层多 Key 多模型,不绑死单一供应商 |
